Display & Position no CSS

Exemplos práticos das principais propriedades

Position: fixed

📌 Position: fixed no men flutuante do site → O menu usa position: fixed;, o que faz com que ele fique preso à tela, mesmo quando a página rola (scroll). Com top: 0; ele fica colado no topo, width: 100%; faz ocupar toda a largura da tela, e z-index: 1000; garante que ele fique na frente dos outros elementos.

Display: block

📌 Display: block → O elemento ocupa toda a largura disponível da tela e sempre começa em uma nova linha. Mesmo que tenha espaço ao lado, ele “quebra” a linha automaticamente. Permite definir largura, altura, margem e padding normalmente.

Display: inline

📌 Display: inline → O elemento fica na mesma linha que outros elementos, ocupando apenas o espaço do seu conteúdo. Não aceita largura e altura diretamente. Ideal para textos ou pequenos elementos lado a lado.

Display: inline-block

📌 Display: inline-block → Mistura o comportamento do inline com o block. Fica na mesma linha que outros elementos, mas permite definir largura e altura. Muito usado para criar botões e cards alinhados.

Display: flex

📌 Display: flex → Ativa o modo Flexbox. Organiza os elementos filhos em linha (ou coluna) de forma flexível, facilitando alinhamento, espaçamento e distribuição automática dos itens dentro do container.

📌 Position → Controla como o elemento é posicionado na página. Pode ser estático (padrão), relativo, absoluto, fixo ou sticky, alterando como ele se comporta em relação ao fluxo normal do documento.

Position

📌 Position: relative → O elemento continua ocupando seu espaço normal na página, mas pode ser deslocado usando top, bottom, left ou right. Também serve como referência para elementos com position: absolute dentro dele.

📌 Position: absolute → O elemento sai do fluxo normal da página e é posicionado em relação ao elemento pai que tenha position diferente de static. Pode ser colocado exatamente onde você quiser usando top, right, bottom e left.

📌 Position: sticky → O elemento se comporta como relativo até atingir um limite (definido com top, por exemplo). Depois disso, ele “gruda” na tela durante o scroll.
